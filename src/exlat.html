Why does ExaLAT care about GPUs?

  • Massively parallel performance
    • Good at BLASy type operations
    • Established hardware available in many places
    • ...
    • profit
  • Why wouldn't you want GPUs
    • CUDA-based programming a low but not zero bar to entry
    • Not "open" and not necessarily available on Intel/AMD hardware

I'm not into lattice...

  • My view:
    • It's mostly CG
    • Really big CG
    • With more maths added on later
  • How to exploit the available resources:
    • Make sure you can decompose problem across GPUs
    • Ideally, decomposable across threads
    • Minimise halo swapping
  • Ideal code can adapt to use N GPUs at runtime
    • You don't want to have to tweak code for each set of resources
    • Multiple ways to do this
    • ... of escalating complexity

AtomicAdd

  • Enter, your new best friend, AtomicAdd...
  • Forces the threads to behave sensibly by adding their component in order
  • Probably not the fastest method to do this, but it is a CUDA function, so likely to be as fast as possible 


	v_idx = threadIdx.x;
	atomicAdd(&temp, tempa[v_idx]);
